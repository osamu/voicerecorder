# PRD.md — 自動クラウドアップロード型ボイスレコーダ プロダクト要件定義

> 本書はレビュー（`REVIEW.md`）反映後の**正の仕様**。当初インタビュー要件のうちレビューで上書きされた項目は「確定要件」の内容が優先する。技術詳細は `DESIGN.md` を参照。

## 1. コンセプト

**「どんな会議でも、録音すれば自動でクラウド（Google Drive）へ上がるボイスレコーダ」**

録音停止と同時に音声が Drive のアプリ管理フォルダへ自動アップロードされ、録音後の非同期文字起こしテキスト（`.txt`）も同じ場所に別ファイルとして届く。ユーザーは「録音ボタンを押して、止める」以外の操作をしない。

## 2. 解決するペイン

- ボイスメモは録れるが、会議後に音声や文字起こしを手動で他ツールへコピペ・アップロードする後処理が面倒。
- 録音資産がローカルに散在し、要約・議事録生成・タスク抽出などの自動 Workflow に繋がらない。
- → Drive の固定フォルダ（`/VoiceRecorder/`）に自動で溜まる構造にすることで、Drive をトリガーにした後段 Workflow へ直結できる。

## 3. ターゲット

- 一次: 開発者本人（個人利用）。対面会議・打ち合わせを日常的に録音し、Drive ベースの自動処理パイプラインを持つ/作るユーザー。
- リモート会議はスピーカー出力経由の集音でカバー（他アプリの内部音声取得は OS 制約により対象外）。
- ストア公開は将来課題（本 MVP は個人利用割り切り）。

## 4. 確定要件

### 4.1 プラットフォーム / フォーマット

- iOS + Android、Flutter 単一コードベース。**Android は `minSdkVersion 29`**。
- 音声: **iOS = AAC `.m4a`（モノラル 32kbps）／ Android = Ogg Opus（`.opus`）。OS 間の非統一を許容**（将来 libopus FFI 統合で `.opus` 統一を検討）。音質設定は出さない。
- ※当初の「両 OS 共通 `.opus`」「録音中リアルタイム文字起こし」「Drive フルアクセス＋任意フォルダ指定」はレビューにより撤回済み。

### 4.2 録音

- 音声ソースは端末内蔵マイクのみ（固定）。
- **バックグラウンド／画面ロック中も録音継続**。録音開始は必ずフォアグラウンドから（Android 11+ 制約）。
- 録音中の表示: Android = FGS 常駐通知に経過時間 / iOS = OS 標準マイクインジケータ＋アプリ内録音中バー。
- 同時録音は 1 本のみ（多重録音排他）。一時停止は MVP 非対応。最大録音時間の上限なし（空き容量チェックで実質制限）。
- **データ保全（必須）**: 逐次書き込み＋定期フラッシュ、割り込み（着信/Siri）時は確定保存→自動再開→別セグメント（`_part2`）、異常終了後の起動時リカバリ（「中断された録音を復元しました」）、開始時・録音中の空き容量チェックと枯渇時の安全クローズ。

### 4.3 アップロード（Google Drive）

- 認証: 単一 Google アカウント、Google Sign-In。スコープは **`drive.file` のみ**。
- 保存先: **アプリが自作するルートフォルダ `/VoiceRecorder/` 配下のみ**。任意の既存フォルダ指定は不可。配下に日付サブフォルダ `<年>/<年-月>` を自動生成（録音開始時点の端末ローカル時刻基準）。
- タイミング: 録音停止直後に自動アップロード（フォアグラウンド中は即時）。オフライン・失敗時はキューに積み、ネット復帰・アプリ再起動・FG 復帰で自動リトライ。バックグラウンドでの実行は OS 裁量のベストエフォート（保証は「次回 FG 復帰までに」）。24 時間以上の滞留はローカル通知。
- **冪等性**: resumable upload、ファイル名は秒まで、ローカル UUID を Drive appProperties に付与し二重作成を防止。改名・削除・txt 紐付けは常に fileId 基準。
- 状態機械: 未アップ → アップ中 → 完了 / 要対応（恒久エラー時。原因別導線＋手動再試行）。一時エラーは指数バックオフで自動リトライ。
- ローカルファイルはアップ後も端末に残す（アプリ内再生用）。**逼迫時の自動削除はアップロード完了済み・古い順のみ**（閾値: 空き容量 500MB 未満・仮値）。未アップ録音は自動削除しない。
- ファイル名: `YYYY-MM-DD_HH-mm-ss[_タイトル].<ext>`、文字起こしは同名 `.txt`。

### 4.4 文字起こし

- 設定で ON/OFF（オプション機能）。
- 方式: **録音完了後にファイルベースで非同期文字起こし。MVP の既定エンジンはクラウド STT（Whisper API 等）**。リアルタイム画面表示は将来オプション（オンデバイス対応言語のみ・ベストエフォート）。
- エンジンは**プラガブル**（出力=イベントストリーム統一、入力=Streaming/Batch 分離、capability＋動的可用性照会。詳細は `DESIGN.md` §8）。将来: OS 標準（リアルタイム）、オンデバイス Whisper 等を追加。
- 言語: 設定で選択（選択中エンジンの対応言語のみ表示。既定は端末ロケール）。
- 出力: プレーンテキスト `.txt`。録音とは別ファイルとして同じ Drive フォルダへアップロード。部分成功時は得られた分を保存し「一部のみ」状態を表示。録音単位の再文字起こしが可能（Drive の txt は同名上書き）。
- **依存関係: 録音アップロードが常に優先、文字起こしはベストエフォート**。文字起こしの失敗・OFF・未対応でも音声ファイルは必ず Drive に上がる。ジョブは永続化され、アプリ再起動を跨いで完走する。

### 4.5 録音一覧・操作

- 起動時に録音一覧を表示。各項目: 録音日時・録音時間・ファイルサイズ・**アップロードバッジ**（未アップ〔未サインイン時は「Drive未設定」〕/ アップ中 / 完了 / 要対応）・**文字起こしバッジ**（OFF / 処理中 / 完了 / 一部のみ / 失敗）。
- 操作: アプリ内再生（シークバー・±15秒・再生速度）/ タイトル変更（タイトル部のみ、Drive 側ファイル名も fileId 基準で追従、音声と txt はペア改名）/ 削除 / 再文字起こし / txt 閲覧。
- 削除は既定でローカルのみ。Drive 側削除は明示チェック＋警告。**未アップ録音の削除は強警告＋キュー除去**。
- 自動削除済み録音の再生は「Drive から再取得（要オンライン）」。
- 一覧上部にオフラインバナー。録音中は全画面共通の録音中バー。

### 4.6 オンボーディング・設定

- **録音はサインイン不要で常に可能**（アップロードのみ保留）。初回起動はマイク権限のみ必須。権限拒否時は説明→OS 設定誘導。
- 設定画面: アカウント連携（サインイン/アウト。サインアウト時は未アップ件数警告・キュー保持）、Drive 保存先表示（固定）、文字起こし ON/OFF・言語・API キー、ストレージ管理。

### 4.7 後段 Workflow 契約

- Workflow は `/VoiceRecorder/` をフォルダ ID で監視。音声ファイル到着をトリガーとする。
- **`.txt` の到着は保証しない**（任意添付）。音声と txt のペアリングはファイル名でなく **appProperties の不変 ID** で行う。

## 5. スコープ外（MVP）

- リアルタイム文字起こし表示（将来オプション。抽象レイヤに席のみ確保）
- 任意既存 Drive フォルダの指定・共有ドライブ・共有フォルダ
- 再インストール時の Drive からの一覧復元
- 多言語混在会議
- 録音の一時停止、Bluetooth 等外部マイク切替、音質設定
- 他アプリ（Zoom/Meet）内部音声の取得
- ストア公開対応一式（同意ダイアログ、プライバシーポリシー URL、App Privacy / Data Safety 申告、Live Activity）

## 6. 非機能要件

### 6.1 データ保全（最重要）

- 「録音すれば必ず Drive に上がる」の前提として、録音データはクラッシュ・電池切れ・割り込み・容量枯渇のいずれでも**直近数秒を超えて失われない**こと（§4.2 の保全要件）。
- 未アップロードデータは自動削除・サインアウト・認証エラーのいずれでも失われないこと（キュー保持）。
- アップロードは冪等（リトライしても Drive 上に重複ファイルを作らない）。

### 6.2 セキュリティ

- OAuth トークンは flutter_secure_storage（iOS Keychain: AfterFirstUnlockThisDeviceOnly / Android Keystore）。サインアウト時に revoke。
- 録音・txt・メタ DB はアプリ内部ストレージ限定＋バックアップ除外。iOS ファイル保護は CompleteUntilFirstUserAuthentication。
- ログにトークン・タイトル・文字起こし本文を出力しない。
- Drive 権限は `drive.file` に最小化（漏洩時の被害範囲をアプリ作成ファイルに限定）。

### 6.3 プライバシ（個人利用割り切り）

- MVP は個人利用前提とし、録音同意ダイアログ等の配慮は**最小限**。録音の適法性（相手の同意取得等）はユーザーの責任に帰属。
- 文字起こし ON 時、音声が外部サーバ（クラウド STT）へ送信される。MVP では設定画面の説明文で開示し、ストア公開時に明示オプトインへ格上げする。
- ストア公開時には、初回同意ダイアログ・利用規約/プライバシーポリシー・両ストアのデータ申告を別途対応する（`TODO.md` Phase 8）。

## 7. 受け入れ基準（MVP 検証項目）

1. iOS / Android 実機で録音→停止→Drive の `/VoiceRecorder/<年>/<年-月>/` にファイルが出現する。
2. バックグラウンド・画面ロック中も録音が継続する（Android は常駐通知に経過時間）。
3. 機内モードで録音停止→キュー滞留（バッジ「未アップ」）→復帰後に自動アップロードされる。
4. 録音中の強制終了後、次回起動で録音が復元され、キューに投入される。
5. 文字起こし ON で `.txt` が別ファイルとしてアップされる。OFF・失敗時も音声は必ず上がる。アプリ再起動を跨いでも文字起こしジョブが完走する。
6. 一覧のアップ・文字起こしバッジが実状態に追従する。
7. リトライを挟んでも Drive 上に同一録音のファイルが重複しない。
8. 未サインインでも録音でき、サインイン後に滞留分がアップされる。
